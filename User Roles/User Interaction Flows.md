# 7. User Interaction Flows

## 7.1 Job Seeker ↔ Hiring Manager Interaction

### Application Submission Flow
1. Job Seeker submits application  
 ↓  
2. Hiring Manager receives notification  
 ↓  
3. Hiring Manager reviews application  
 ↓  
4. Hiring Manager updates status or requests interview  
 ↓  
5. Job Seeker receives status notification  
 ↓  
6. Job Seeker responds or confirms interview  
 ↓  
7. Hiring Manager conducts interview  
 ↓  
8. Hiring Manager makes decision  
 ↓  
9. Job Seeker receives final decision  

### Key Touchpoints
- **Initial Application:** Job Seeker submits, Hiring Manager receives  
- **Status Updates:** Hiring Manager changes status, Job Seeker notified  
- **Messages:** Bidirectional communication for questions/clarifications  
- **Interview:** Coordination and scheduling  
- **Decision:** Offer or rejection communicated to Job Seeker  

## 7.2 Recruiter ↔ Hiring Manager Interaction

### Collaborative Hiring Flow
1. Hiring Manager submits job requisition  
 ↓  
2. Recruiter receives assignment and discusses requirements  
 ↓  
3. Recruiter sources and screens candidates  
 ↓  
4. Recruiter presents shortlisted candidates to Hiring Manager  
 ↓  
5. Hiring Manager reviews and provides feedback  
 ↓  
6. Recruiter coordinates interviews  
 ↓  
7. Hiring Manager interviews and evaluates  
 ↓  
8. Recruiter facilitates offer or continues search  

### Key Collaboration Points
- **Intake Meeting:** Align on role requirements and process  
- **Candidate Review:** Recruiter submits, Hiring Manager evaluates  
- **Feedback Loop:** Continuous communication on candidate quality  
- **Interview Coordination:** Recruiter schedules, Hiring Manager attends  
- **Decision Support:** Recruiter provides market data and guidance  

## 7.3 Company Owner ↔ All Roles

### Governance & Oversight Flow
1. Company Owner sets strategy and policies  
 ↓  
2. Hiring Managers and Recruiters execute hiring  
 ↓  
3. Company Owner monitors via analytics  
 ↓  
4. Company Owner provides guidance and adjustments  
 ↓  
5. Team implements improvements  

### Key Management Functions
- **Team Setup:** Owner invites and assigns roles  
- **Process Standardization:** Owner sets templates and workflows  
- **Performance Monitoring:** Owner reviews metrics and reports  
- **Support & Guidance:** Owner provides resources and training  
- **Strategic Decisions:** Owner adjusts hiring strategy based on data  

## 7.4 System Administrator ↔ All Users

### Support & Maintenance Flow
1. User encounters issue or needs assistance  
 ↓  
2. User reports to support or System Admin investigates proactively  
 ↓  
3. System Admin troubleshoots and resolves  
 ↓  
4. System Admin communicates resolution  
 ↓  
5. System Admin documents for future reference  

### Key Support Scenarios
- **Technical Issues:** System Admin resolves platform bugs  
- **Account Management:** System Admin assists with access and permissions  
- **Data Requests:** System Admin fulfills GDPR/compliance requests  
- **Training & Guidance:** System Admin provides technical documentation  
- **Platform Improvements:** System Admin implements feedback from users
